Анализ журнала отметок РЛС

РЛС периодически сканирует пространство и записывает обнаруженные цели в журнал.
Журнал формируется автоматически и может содержать ошибки (сбои связи, повреждённые строки).
Программа должна устойчиво обрабатывать такой файл и выдавать аналитический отчёт.

Формат входного файла

# time_ms  target_id  range_m  azimuth_deg
1000       3          45200    120.5
2000       3          44800    121.0
2000       7          88100    034.2
3000       3          44350    121.4

- Строки начинающиеся с # — комментарии, игнорировать
- Разделитель — любое количество пробелов или табов
- Поля: time_ms (целое), target_id (целое 1–255), range_m (целое), azimuth_deg (вещественное 0.0–359.9)
- Записи в файле не обязательно упорядочены по времени

Требования к программе

Запуск

./rls_analyze <файл>                        # полный отчёт
./rls_analyze <файл> --sector <az1> <az2>   # только цели в секторе азимутов
./rls_analyze <файл> --top <N>              # N ближайших целей

Аргументы --sector и --top могут комбинироваться.

Коды завершения

0 - Успех
1 - Файл не найден или не читается
2 - Нет ни одной корректной записи
3 - Неверные аргументы командной строки

Валидация входных данных

Некорректные строки не прерывают работу программы. Для каждой такой строки в stderr выводить:
Строка 7: пропущена — неверный формат (ожидалось 4 поля, найдено 2)
Строка 12: пропущена — azimuth_deg=400.0 вне диапазона [0, 360)

После обработки всего файла в stderr итог:
Итого: прочитано 47 строк, принято 43, пропущено 4

Обработка отметок

- Если для одной цели пришли две отметки с одинаковым time_ms — оставить ту, у которой меньше range_m, вторую считать дублем и сообщить в
stderr
- Перед анализом отсортировать отметки каждой цели по time_ms — реализовать сортировку вставками вручную, без qsort

Отчёт

Пример вывода — формат должен совпадать посимвольно (ширина колонок фиксирована):

=== Отчёт по журналу: data.csv ===
Всего целей: 4  |  Всего отметок: 43  |  Дублей: 2

ID  Отметок  Дальн.мин(м)  Дальн.макс(м)  Азимут.ср(°)  Скорость(м/с)
---  -------  ------------  -------------  ------------  -------------
3       12         44350          51200        121.3°        -187.5
7        8         21000          88100         34.8°         +92.0
15        3         67400          71200        280.1°             --
22        1         39800          39800         95.5°             --

Ближайшая цель: ID=7, минимальная дальность 21000 м (время 5000 мс)

Примечание: скорость вычислена только для целей с 3+ отметками.
Знак: "−" — цель удаляется, "+" — цель сближается.

Цели в таблице сортировать по минимальной дальности (по возрастанию) — реализовать сортировку выбором вручную, без qsort.

Если --sector задан, в заголовке добавить строку:
Фильтр: сектор 30.0° — 150.0°  (показаны 2 из 4 целей)


Ограничения реализации

- Язык: строго C (C99), не C++
- Никаких сторонних библиотек — только стандартная библиотека C
- Статические массивы: максимум 64 цели, 1000 отметок — объявить константами #define MAX_TARGETS 64
- Никакого malloc / calloc / realloc
- Код разбить минимум на три файла: main.c, tracks.c, tracks.h

---
Bash-скрипты (обязательны)

gen_test.sh — генератор тестовых данных

./gen_test.sh <число_целей> <отметок_на_цель> <выходной_файл>

- Генерирует корректный CSV
- Намеренно вставляет 3–5 некорректных строк (неверное число полей, значение вне диапазона)
- Намеренно вставляет 2–3 дубля (одинаковые time_ms и target_id)
- Данные генерировать случайно через $RANDOM, но воспроизводимо: первым аргументом принимать seed (необязательный 4-й аргумент)

run_tests.sh — автоматическая проверка

Скрипт запускает программу на нескольких сценариях и проверяет корректность кодов завершения:

# Примеры проверок внутри скрипта:
./rls_analyze несуществующий_файл
[ $? -eq 1 ] || echo "FAIL: ожидался код 1"

./rls_analyze пустой.csv
[ $? -eq 2 ] || echo "FAIL: ожидался код 2"

./rls_analyze data.csv --top abc
[ $? -eq 3 ] || echo "FAIL: ожидался код 3"

./rls_analyze data.csv --top 2 | grep "Ближайшая цель"
[ $? -eq 0 ] || echo "FAIL: строка не найдена в выводе"

Скрипт должен в конце вывести Все тесты пройдены или перечислить провалившиеся.
